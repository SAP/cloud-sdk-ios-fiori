PRIVACY STATEMENT

Protecting the individual's privacy is crucial to the future of business. We have created this Privacy Statement to demonstrate our firm commitment to the individual`s right to data protection and privacy. This Privacy Statement outlines how we handle information that can be used to directly or indirectly identify an individual.

 

A. General information

 

Who is the Data Controller?

The data controller for SAP Mobile Start is SAP SE, Dietmar-Hopp-Allee 16, 69190 Walldorf, Germany (SAP) and its data protection officer can be reached at privacy[@]sap.com.

 

What Personal Data does SAP collect?

SAP collects some information about you in the context of your professional activities, consisting of username, email address, usage tracking data, recorded voice, account information (Personal Data).

 

Why does SAP need your Personal Data?

SAP requires your Personal Data to:  

 

-       authenticate & authorize the user against the Identity provider

During the onboarding flow, the user is redirected to the configured identity providers login page to authenticate, using the appropriate credentials. The credentials are checked in the connected infrastructure.

 

-        display push notifications

Push notification messages can be sent from the server (Mobile Services tenant). For this purpose, the server persists a local registry of users and tokens, which allows the correct dispatching of these messages.

 

-       enable app communication with the backend services

SAP Mobile Start app requires the Mobile Services tenant URL and the SiteID information to communicate with the server. This information is retrieved from the QR code used in the on-boarding process.

 

-       suggest search terms for the in-app search

Based on past searches, the in-app search can display a list of previously used terms, for better user experience. This is achieved by persisting locally the search terms and the proper timestamps.

 

 

 

-       use voice for the in-app search

Leveraging the Apple iOS voice-to-text capabilities, the user can activate this feature in the SAP Mobile Start app to perform a search using the voice instead of typing the search term.

 

-       suggest available tiles in Spotlight search

Using the iOS Spotlight service, a user can search for any tile that is available in SAP Mobile Start app, without opening the app itself. This is achieved by persisting in the local index of the Spotlight service a list of tile titles and subtitles (e.g. Sales Volume, Sales Contracts etc.).

 

SAP Mobile Start is processing business data from the following 5 roles as well as launching respective S/4 Fiori Applications:

       Project manager

       Warehouse Clerk

       Finance Manager

       Sales Manager

 

The app also requires the following permissions:

-       The feature for registering your device requires access to the camera and/or photo gallery, in order to read the QR code to start the registration.

-       The feature for using voice search requires access to the device microphone.

 

Although providing Personal Data is voluntary, without your Personal Data SAP cannot provide you with access to the SAP Mobile Start.

How will SAP verify requests to exercise data protection rights?

SAP will take steps to ensure that it verifies your identity to a reasonable degree of certainty before it will process the data protection right you want to exercise. When feasible, SAP will match Personal Data provided by you in submitting a request to exercise your rights with information already maintained by SAP. This could include matching two or more data points you provide when you submit a request with two or more data points that are already maintained by SAP.

SAP will decline to process requests that are manifestly unfounded, excessive, fraudulent, or are not otherwise required by local law.
